# Final Verification Notes For Product Overview

## Область проверки

Финальная проверка была выполнена для файла `docs/product-overview.md` с опорой на:

- код в `main`;
- профильные тесты;
- recent merged commits;
- документы в `_bmad-output` как secondary context;
- GitHub Project `bon2362/3` только как cross-check для будущих направлений.

## Что проверено и подтверждено

### Shipped behavior, который безопасно описан в overview

- Локальная библиотека книг.
- Импорт `EPUB` и `PDF`.
- Drag-and-drop импорт книг.
- Поиск по библиотеке с подсветкой совпадений.
- Чтение EPUB и PDF.
- Оглавление, поиск по книге и переход к странице.
- Восстановление позиции чтения.
- Highlights, text notes, sticky notes.
- Панель аннотаций по текущей книге.
- Экспорт аннотаций текущей книги.
- Массовый экспорт аннотаций по библиотеке.
- Импорт Markdown-аннотаций с preview.

## Что было специально исключено или ослаблено

### Не заявлено как shipped

- iCloud sync.
- iPhone-версия.
- Настройки темы, шрифта и размера текста.
- Автоматический фоновый экспорт аннотаций.
- Дополнительные форматы beyond `EPUB` / `PDF`.

### Заявлено с осторожностью

- TXT export не назван shipped-возможностью.
  Причина: в current code уверенно подтверждён Markdown exchange flow, а отдельный пользовательский TXT flow не найден как канонический shipped behavior.

## Найденные риски unsupported claims

1. Старые документы содержат устаревшие утверждения.
   В частности, ранние PRD и retrospective-материалы уже не отражают текущий shipped status PDF и import/export аннотаций.

2. В проектных материалах есть будущие направления, которые легко перепутать с текущим состоянием.
   Это особенно касается sync, iPhone и настроек чтения.

3. Исторические формулировки про движок EPUB и roadmap нельзя напрямую переносить в user-facing обзор.
   Поэтому финальный overview вообще не опирается на архитектурные обещания там, где достаточно описать пользовательский эффект.

## Итоговое решение

`docs/product-overview.md` соответствует текущему состоянию репозитория на уровне user-facing возможностей и явно отделяет future / planned направления от уже shipped behavior.

Критичных unsupported claims после финального прохода не оставлено.
