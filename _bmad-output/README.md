# _bmad-output

Артефакты BMAD теперь организованы по области применения, а не только по типу документа.

## Структура

- `project-docs/` — общепроектные документы: PRD, архитектура, список эпиков, ретроспектива MVP, статус спринта.
- `feature-docs/<feature-slug>/` — материалы конкретной фичи. Внутри папки используются короткие имена файлов, потому что контекст уже задан именем папки.
- `stories/` — отдельные story-файлы в плоской структуре, но с именами, которые сразу показывают эпик и story.
- `test-artifacts/` — место для тестовых артефактов, если они появятся позже.

## Нейминг

- Общепроектные документы: `<artifact>-reader-app[-scope].md`
- Фича-документы: `feature-docs/<feature-slug>/plan.md`, `implementation-spec.md`, `retrospective.md`
- Story-файлы: `epic-<n>-<epic-slug>__story-<n.m>-<story-slug>.md`

## Принципы

- Сначала видно контекст: проект, фича или story.
- Материалы одной фичи лежат рядом, даже если один документ про план, а другой про реализацию.
- Имя файла должно быть понятно вне Finder preview: по нему сразу видно, к чему относится документ.
