# Reader: продуктовый обзор

## 1. Что такое Reader

`Reader` — это нативное macOS-приложение для личного чтения книг в форматах EPUB и PDF. Оно сочетает локальную библиотеку, режим чтения с сохранением позиции и систему аннотаций, чтобы пользователь мог читать, выделять фрагменты, писать заметки и возвращаться к ним позже.

Это локальное приложение: книги, прогресс чтения и аннотации хранятся на устройстве пользователя. В текущей версии `Reader` сфокусирован на спокойном рабочем сценарии "добавил книгу -> читаю -> оставляю пометки -> при необходимости экспортирую или импортирую аннотации".

## 2. Для кого приложение

`Reader` рассчитан на пользователя, которому нужна личная библиотека и удобное чтение без облачного сервиса и без лишней инфраструктуры. Приложение особенно подходит для сценариев, где важно:

- держать книги локально на Mac;
- читать и EPUB, и PDF в одном приложении;
- делать highlights, заметки по тексту и sticky-заметки по страницам;
- возвращаться к книге точно на том месте, где чтение было остановлено;
- выгружать аннотации в переносимый Markdown-формат и при необходимости импортировать их обратно.

## 3. Что пользователь делает в приложении

Обычный пользовательский цикл в `Reader` выглядит так:

1. Добавить EPUB или PDF в библиотеку через диалог выбора файла или простым перетаскиванием в окно библиотеки.
2. Найти нужную книгу по названию или автору.
3. Открыть книгу и читать её постранично.
4. Пользоваться оглавлением, поиском по книге и переходом к конкретной странице.
5. Создавать highlights, текстовые заметки и sticky-заметки.
6. Открывать панель аннотаций и быстро возвращаться к нужному месту в книге.
7. При необходимости экспортировать аннотации текущей книги или всей библиотеки в Markdown.
8. При необходимости импортировать ранее сохранённые Markdown-аннотации через режим preview.

## 4. Устройство приложения с точки зрения пользователя

С точки зрения интерфейса приложение состоит из двух основных уровней:

- `Библиотека`
  Главный экран с карточками книг. Здесь пользователь добавляет книги, удаляет их, ищет по библиотеке и запускает библиотечный импорт или экспорт аннотаций.
- `Экран чтения`
  Отдельный режим для открытой книги. Здесь доступны само чтение, оглавление, поиск по книге, панель аннотаций, sticky-заметка и экспорт аннотаций текущей книги.

В библиотеке карточка книги показывает:

- обложку или заглушку;
- название;
- автора, если он найден;
- индикатор прогресса, если чтение уже началось;
- badge формата (`EPUB` или `PDF`).

Одинарный клик выделяет карточку, двойной открывает книгу. Удаление книги запускается из выделенной карточки и подтверждается отдельным диалогом.

## 5. Основные возможности

Текущая версия `Reader` уже поддерживает:

- локальную библиотеку EPUB и PDF;
- импорт книг через системный диалог выбора файла;
- drag-and-drop импорт книг прямо в библиотеку;
- поиск по библиотеке по названию и автору;
- подсветку совпадений в найденных карточках;
- чтение EPUB;
- чтение PDF;
- сохранение позиции чтения и повторное открытие книги на последнем месте;
- highlights, текстовые заметки и sticky-заметки;
- правую панель аннотаций с быстрым переходом к месту в книге;
- экспорт аннотаций текущей книги;
- экспорт аннотаций всей библиотеки;
- импорт Markdown-аннотаций с предварительным предпросмотром;
- переход к конкретной странице через индикатор страницы.

## 6. Работа с EPUB

Для EPUB в `Reader` доступен полноценный режим чтения:

- постраничное чтение;
- переходы вперёд и назад клавишами, пробелом и кликами по краям страницы;
- боковая панель оглавления;
- поиск по всей книге;
- список недавних поисковых запросов;
- переход по внутренним ссылкам;
- возможность вернуться назад после перехода по ссылке;
- переход к конкретной странице через индикатор внизу экрана.

Верхняя часть экрана показывает текущий раздел книги, а нижний индикатор отображает текущую страницу и общее число страниц. Если пользователь нажимает на индикатор, он может ввести номер страницы и сразу перейти к ней.

При повторном открытии EPUB книга возвращается к последней сохранённой позиции. Приложение также кэширует информацию о числе страниц по главам, чтобы повторное открытие было быстрее и не требовало пересчёта на каждом запуске.

## 7. Работа с PDF

Для PDF предусмотрен отдельный поток чтения, но с близкой пользовательской логикой:

- открытие PDF из той же библиотеки;
- восстановление позиции при повторном открытии;
- постраничная навигация;
- переход к конкретной странице через индикатор страницы;
- оглавление на основе outline PDF, если оно доступно в файле;
- поиск по PDF;
- переход к найденным результатам поиска;
- переход к аннотациям и обратно.

Важно учитывать ограничение: поиск в PDF работает только там, где у документа есть текстовый слой. Для image-only PDF приложение показывает, что поиск недоступен.

## 8. Аннотации и заметки

В `Reader` сейчас есть три основных типа пользовательских пометок:

### Highlights

Пользователь выделяет текст и выбирает один из пяти цветов:

- жёлтый;
- красный;
- зелёный;
- синий;
- фиолетовый.

Существующий highlight можно открыть повторно, поменять ему цвет или удалить его.

### Заметка к тексту

Из выделения можно сразу создать текстовую заметку. Это подходит для случаев, когда нужно не только подсветить фрагмент, но и оставить собственный комментарий рядом с конкретным местом текста.

### Sticky-заметка к странице

Sticky note создаётся для текущей страницы. Это отдельная заметка, не обязательно привязанная к выделенному слову или фразе. Такой сценарий полезен, когда нужно оставить комментарий ко всей странице или развороту чтения.

### Панель аннотаций

Справа в reader view можно открыть панель аннотаций. Она показывает все пользовательские пометки текущей книги и поддерживает вкладки:

- всё;
- хайлайты;
- заметки;
- стикеры.

Нажатие на элемент панели переводит пользователя к соответствующему месту в книге. Для sticky notes интерфейс показывает человекочитаемую привязку к странице.

## 9. Импорт и экспорт

В текущей версии приложение поддерживает не только импорт книг, но и перенос аннотаций.

### Импорт книг

Поддерживаются:

- `EPUB`;
- `PDF`.

Импорт можно запустить двумя способами:

- через кнопку добавления книги;
- через drag-and-drop прямо в библиотеку.

### Экспорт аннотаций

Поддерживаются два пользовательских режима:

- экспорт аннотаций текущей книги из reader view;
- экспорт аннотаций всей библиотеки из экрана библиотеки.

Экспорт создаёт отдельные Markdown-файлы. Если у книги нет аннотаций, приложение сообщает, что экспорт для неё не требуется.

### Импорт аннотаций

Импорт аннотаций запускается из библиотеки. Перед применением пользователь получает предпросмотр, где видно:

- сколько аннотаций будет создано;
- сколько обновится;
- сколько будет пропущено;
- есть ли невалидные файлы;
- не остались ли файлы, для которых книга не найдена.

Сопоставление импортируемых Markdown-файлов с книгами опирается на `contentHash`, а обновление существующих аннотаций — на устойчивые `exchangeId`.

## 10. Сохранение состояния и локальное хранение

Текущая версия `Reader` использует локальное хранение. Это значит:

- книги сохраняются локально и доступны приложению без внешнего сервиса;
- прогресс чтения записывается локально;
- highlights, заметки и sticky notes сохраняются локально;
- повторное открытие книги восстанавливает последнюю известную позицию чтения.

Пользовательский смысл этого подхода простой: приложение работает как личный локальный reader, в котором библиотека и заметки принадлежат устройству пользователя, а не облачному аккаунту.

## 11. Ограничения текущей версии

Ниже перечислены ограничения, которые важно считать частью текущего состояния продукта, а не багами описания.

- Версия ориентирована на `macOS`; мобильной версии в текущем продукте нет.
- Хранение локальное; встроенной облачной синхронизации сейчас нет.
- В PDF поиск зависит от наличия текстового слоя и не работает для image-only документов.
- Поддержка импортируемых книг ограничена форматами `EPUB` и `PDF`.
- В текущем пользовательском UI нет настроек темы, шрифта, размера текста и других параметров внешнего вида чтения.
- В shipped интерфейсе нет автоматического фонового экспорта аннотаций в выбранную папку.
- Не стоит считать отдельный TXT-экспорт подтверждённой возможностью: текущий код надёжно подтверждает именно Markdown exchange flow.

## 12. Ближайшие направления развития

Ниже перечислены направления, которые видны в проектных артефактах и GitHub Project, но не должны восприниматься как уже существующие функции.

### Future / planned

- `iCloud sync` для библиотеки, прогресса чтения и аннотаций.
- `iPhone`-версия, причём в текущих проектных материалах она описана как PDF-first MVP.
- Настройки внешнего вида чтения: тема, шрифт, размер текста и связанные параметры EPUB layout.
- Дополнительные форматы книг beyond EPUB/PDF.
- Более автоматизированный экспорт аннотаций, включая идею постоянного экспорта в выбранную папку.
- Дополнительные сценарии организации библиотеки, например группировка по прогрессу чтения.

Эти направления полезно держать в этом документе как ориентир для будущих обновлений, но они должны оставаться явно отделёнными от shipped-функций, пока их не подтвердят код в `main`, тесты и пользовательский UI.
